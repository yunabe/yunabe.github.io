Linux におけるソケット数 (TCPのコネクション数) の上限¶

ulimit によるファイルディスクリプタ数制限¶

ユーザ毎のファイルディスクリプタ数の上限を超えてソケットを作る (TCPコネクションを張る) ことはできません。 この値は ulimit -n で確認できます。

システム全体のファイルディスクリプタ数の制限¶

システム全体のファイルディスクリプタ数の上限が存在します。

cat /proc/sys/fs/file-max
